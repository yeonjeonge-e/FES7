1. 배열 (Array)
- 데이터를 순서대로 저장하는 객체

<특징>
1. 빈 배열로 생성하거나 요소가 포함된 배열로 생성 할 수 있음
2. 배열은 숫자를 사용하여 값에 접근할 수 있음
  - 숫자는 [값]의 [순서]를 의미
  - 순서 : 인덱스(index)
  - 배열안에 존재하는 값 : 원소 (elements)
3. 원소의 길이를 나타내는 length 프로퍼티를 가지고 있음
4. 배열 안에 다른 배열 포함 가능 -- 다차원 배열 (2차원, 3차원)

<종류>
push() / pop()


  노드 : 브라우저에 랜더링 되는 모든 요소 (띄어쓰기/줄바꿈 등등)

  map() : 배열 순환 후 새로운 배열을 반환 (값을 제공한다)
  forEach() : 배열 순환 - 배열 순환만 하고 끝 & 멈출 수 없음
              - 안에 break문이나 다른 

reduce() : 
include() : 배열 안에 특정한 요소가 포함되어 있는지 아닌지를 확인



2. 객체 (Object)
- 배열처럼 여러개의 데이터를 한 변수에 저장할 수 있는 자료형
- 순서로 접근하지 않고 특별한 키를 통해 원하는 값에 접근할 수 있는 키-값 쌍으로 이루어져 있음
- 리터럴 : 

배열 []
객체 {} 안에는 키와 값이 있는데 그걸 합쳐서 어트리뷰트라고 부른다

함수를 가지는 어트리뷰ㅌ : 메소드

객체{ 키, 값} -> 어트리뷰트 
어트리뷰트 중 값을 가진게 프로퍼티
            함수를 가진게 메서드

객체 안에 있는 함수를 메서드라고 부름

배열 안 원소에 
어트리뷰트
객체 이름에 .연산자


for (변수 in 반복작업을_수행할_객체 ) {

}

- 반드시 순서대로 반복되지 않음
- 순서가 중요한 경우? - for in문 사용하면 안됨
- 프로퍼티 안에 있는 값을 순환하면서 값을 꺼내거나, 변경할 때 주로 사용
- 제거하고 싶을 때는 delete 키워드 사용

객체 안에서 값을 변환하려고 하면 안됨
- 순서가 중요하면 배열을 사용해야 함
- 배열에 쓰여야 할 거는 순서 중요
